# ClawFeed - 深度解读

## 1. 核心洞察 (Insight)

**核心观点**：ClawFeed 是 OpenClaw 生态的内容摄入层——从数千源到精华摘要的 AI 策展系统。

**关键洞见**：
- **多频摘要**：4小时/日/周/月 不同粒度
- **源包（Source Packs）**：可分享的兴趣包
- **零 API key 设计**：FxTwitter、Nitter 等免 key 方案

> "Stop scrolling. Start knowing."

## 2. 架构分析 (Architecture)

### 功能矩阵
| 功能 | 说明 |
|------|------|
| 多频摘要 | 4h / daily / weekly / monthly |
| 源系统 | Twitter、RSS、HN、Reddit、GitHub Trending |
| 源包 | 社区分享的兴趣包 |
| 深度阅读 | Mark → AI 分析 |
| 智能策展 | 可配置噪声过滤 |
| 关注建议 | 基于质量分析 |
| 输出订阅 | RSS / JSON Feed 导出 |

### 技术架构
```
Sources (多平台)
    ↓
Curation Rules (可配置)
    ↓
AI Digest Generation
    ↓
Multi-format Output (Web / RSS / JSON)
```

### 部署选项
1. **ClawHub**（推荐）：`clawhub install clawfeed`
2. **OpenClaw Skill**：克隆到 `~/.openclaw/skills/`
3. **Zylos Skill**：克隆到 `~/.zylos/skills/`
4. **Standalone**：独立部署

## 3. 关键方法论 (Methodology)

### 策展规则定制
- 编辑 `templates/curation-rules.md` 控制过滤逻辑
- 编辑 `templates/digest-prompt.md` 定制 AI 输出格式

### 多租户设计
- SQLite 存储
- Google OAuth 多用户
- 个人书签和源隔离

## 4. 关键数据点 (Data)

### API 端点
| 类别 | 端点 | 说明 |
|------|------|------|
| Digests | GET /api/digests | 列表查询 |
| Auth | GET /api/auth/google | OAuth 流程 |
| Marks | POST /api/marks | 添加书签 |
| Sources | POST /api/sources | 创建源 |
| Packs | POST /api/packs | 创建源包 |
| Feeds | GET /feed/:slug.rss | RSS 订阅 |

## 6. 批判性思考 (Critique)

### 优点
- 完善的安装选项（4种方式）
- 社区分享机制（源包）
- 多语言支持（中英）
- 完整的管理界面

### 局限
- 依赖第三方 API（FxTwitter 等）的稳定性
- 中文内容源支持待验证
- 大规模部署的性能数据未公开

### 改进建议
- 引入个性化学习（用户反馈）
- 添加内容去重机制
- 支持更多国内平台（微信、知乎）
