# gq3171/aobot: 多渠道 AI 网关

## 一句话概述

aobot 是一个基于 pi-agent-rs SDK 构建的多渠道 AI 网关，提供 CLI 聊天、WebSocket JSON-RPC 网关、多 Agent 管理、可扩展的渠道插件框架和持久化存储。

## 项目定位

**目标用户**:
- 需要自建 AI Agent 网关的开发者
- 希望集成多平台（Telegram、Discord）的工程师
- 寻求可扩展 AI 聊天基础设施的团队

**解决的问题**:
- **多平台集成**: 需要同时支持多个聊天平台
- **Agent 管理**: 多 Agent 配置和切换复杂
- **可扩展性**: 难以添加新的渠道支持
- **持久化**: 会话数据难以保存和恢复
- **实时通信**: 需要 WebSocket 实时交互

**使用场景**:
- 多平台 AI 聊天机器人
- 企业内部 AI 网关
- 可扩展的 Agent 基础设施
- 跨平台消息同步

**与同类项目差异**:
- **基于 pi-agent-rs**: 使用成熟的 Rust Agent SDK
- **外部插件系统**: 支持子进程 + NDJSON JSON-RPC 协议的外部插件
- **多协议支持**: CLI、WebSocket、多平台渠道
- **热重载配置**: 文件监控实现配置实时更新
- **自动上下文压缩**: 基于 Token 的压缩和增量 LLM 摘要

## README 中文简介

**aobot** - 基于 pi-agent-rs SDK 的多渠道 AI 网关

**功能**:

**CLI Chat**:
交互式终端聊天，支持可配置的 AI Agent

**WebSocket Gateway**:
- JSON-RPC 2.0 服务器 over WebSocket
- Bearer Token 认证

**多 Agent 管理**:
命名 Agent 配置，支持不同模型、系统提示和工具

**渠道插件框架**:
可扩展集成点，支持外部平台（Telegram、Discord 等）

**外部插件系统**:
- 运行时加载的渠道插件 via 子进程 + NDJSON JSON-RPC 协议
- 支持用任何语言编写插件，无需重新编译 aobot

**插件 SDK**:
Rust crate (aobot-plugin-sdk) 用于快速构建外部渠道插件

**内置渠道**:
- **Telegram 渠道**: 内置 Telegram 机器人集成，支持长轮询、消息分割和内联键盘
- **Discord 渠道**: 内置 Discord 机器人集成

**高级功能**:

**自动上下文压缩**:
- 基于 Token 的压缩
- 结构化序列化
- 增量 LLM 摘要（与 pi-mono 对齐）

**自动重试**:
指数退避处理瞬态 API 错误（速率限制、5xx、网络错误）；上下文溢出通过压缩处理

**配置热重载**:
通过 ~/.aobot/config.toml 文件监控实现实时配置更新

**持久化存储**:
基于 SQLite 的会话元数据持久化；会话在网关重启后保留

**工作区结构**:
```
crates/
```

## 运行与开发方式

**安装**:
```bash
# 从源码构建
git clone https://github.com/gq3171/aobot.git
cd aobot
cargo build --release
```

**配置**:
```toml
# ~/.aobot/config.toml
[agents.default]
model = "gpt-4"
system_prompt = "You are a helpful assistant"
```

**运行 CLI**:
```bash
./target/release/aobot-cli
```

**运行网关**:
```bash
./target/release/aobot-gateway
```

**开发插件**:
使用 `aobot-plugin-sdk` crate 开发外部渠道插件

## 外部接口

**WebSocket API**:
- 协议: JSON-RPC 2.0
- 认证: Bearer Token
- 端点: WebSocket

**CLI 命令**:
| 命令 | 功能 |
|------|------|
| `chat` | 交互式聊天 |
| `agent list` | 列出 Agent |
| `agent switch <name>` | 切换 Agent |
