## 一句话概述

基于WezTerm深度定制的macOS终端模拟器，专为AI辅助编程场景优化，提供开箱即用的默认配置、显著的性能提升和内置AI命令修复助手。

## 项目定位

Kaku定位为面向开发者的高性能macOS终端，重点解决AI编程工作流中的命令行效率问题。作为WezTerm的深度定制分支，它在保持Lua配置兼容性的同时，通过激进的功能裁剪和性能优化，实现了40%更小的二进制体积和更快的启动速度。

## README中文简介

Kaku是一个深度定制的WezTerm分支，旨在提供开箱即用的终端体验。核心特性包括：零配置默认设置（JetBrains Mono字体、macOS字体渲染）、主题自动感知（随系统深浅模式切换）、内置zsh插件套件、40%更小的二进制体积、完整WezTerm Lua配置兼容、以及内置AI助手用于命令错误恢复。

项目还集成了Kaku AI功能，支持VivGrid、MiniMax、OpenAI等多种AI提供商，可自动分析失败的命令并提供修复建议。用户可通过快捷键Cmd+Shift+E一键应用AI建议。

## 运行与开发方式

**安装方式**：
- DMG安装：下载最新release，拖拽至Applications
- Homebrew：`brew install tw93/tap/kakuku`

**首次启动**：
1. 运行应用，Kaku会自动设置Shell环境
2. 使用命令 `/Applications/Kaku.app/Contents/MacOS/kaku init --update-only && exec zsh -l`
3. 运行 `kaku doctor` 检查配置

**开发/配置**：
- 配置文件：`~/.config/kaku/kaku.lua`
- CLI命令：`kaku ai`、`kaku config`、`kaku doctor`
- 文档目录：`docs/`（keybindings.md, features.md, configuration.md, cli.md, faq.md）

**快捷键示例**：
- Cmd+T: 新建标签页
- Cmd+Shift+A: 打开AI面板
- Cmd+Shift+E: 应用AI建议
- Cmd+Shift+G: 打开Lazygit

## 外部接口

**命令行接口**：
- `kaku init --update-only` - 初始化/更新环境
- `kaku doctor` - 诊断检查
- `kaku ai` - AI设置管理
- `kaku config` - 配置管理

**AI Provider API**：
- VivGrid (默认): `https://api.vivgrid.com/v1` - DeepSeek-V3.2
- MiniMax: `https://api.minimax.io/v1` - M2.7, M2.5系列
- OpenAI: `https://api.openai.com/v1` - 自动检测模型
- Custom: 手动配置

**快捷键系统**：完整的键盘绑定通过Lua配置暴露，支持自定义修改。

## 数据流/控制流

**启动流程**：
```
Application Launch → kaku init → Shell Environment Setup 
→ JIT Initialization → Window/Pane Ready
```

**AI修复流程**：
```
User Command → Shell Execute → Error Detected 
→ AI Analysis Request → Suggestion Generated 
→ User Review (Cmd+Shift+E) → Apply Fix / Ignore
```

**配置加载**：
```
~/.config/kaku/kaku.lua → Lua Parser → WezTerm Config API 
→ Theme/Keybindings Applied
```

## 成熟度与风险

**成熟度指标**：
- 开发状态：活跃维护（有持续commit和release）
- 用户规模：README有贡献者展示和赞助页面，表明有一定社区
- 文档完整：提供完整的keybindings、features、configuration、CLI、FAQ文档

**已知限制**：
- 平台锁定：仅支持macOS，Windows/Linux用户无法使用
-上游依赖：深度依赖WezTerm，需跟进上游更新
- AI功能依赖外部API，网络可用性影响功能

**潜在风险**：
- WezTerm API变更可能导致配置兼容性问题
- Apple签名/公证要求可能影响自定义构建分发
- AI provider依赖可能导致厂商锁定
