# tobi/qmd: 本地设备文档搜索引擎

## 一句话概述

QMD 是一个本地设备上的搜索引擎，用于索引 Markdown 笔记、会议记录、文档和知识库，支持关键词搜索和自然语言语义搜索，结合 BM25 全文搜索、向量语义搜索和 LLM 重排序，专为 Agent 工作流设计。

## 项目定位

**目标用户**:
- 需要本地搜索大量 Markdown 笔记和文档的知识工作者
- 使用 AI Agent 并需要上下文检索能力的开发者
- 关注隐私、不想将文档上传到云端搜索服务的用户
- 需要语义搜索而非简单关键词匹配的高级用户

**解决的问题**:
- **笔记搜索困难**: 大量 Markdown 笔记难以快速检索
- **语义理解缺失**: 传统搜索无法理解自然语言查询的意图
- **云端隐私顾虑**: 不想将敏感文档上传到第三方搜索服务
- **Agent 上下文检索**: AI Agent 需要高效获取相关文档上下文

**使用场景**:
- 本地知识库语义搜索
- 会议记录快速检索
- Agent 工作流的文档上下文提供
- MCP 服务器集成 Claude Desktop

**与同类项目差异**:
- **本地优先**: 所有索引和搜索在本地运行，使用 node-llama-cpp 和 GGUF 模型
- **混合搜索**: BM25 + 向量语义搜索 + LLM 重排序三层架构
- **Agent 原生**: --json 和 --files 输出专为 AI Agent 设计
- **MCP 协议支持**: 内置 MCP 服务器，支持 Claude Desktop 和 Claude Code 集成

## README 中文简介

**QMD** - Query Markup Documents

**核心理念**: 为你的记忆提供本地设备搜索引擎。

**核心能力**:
- 索引 Markdown 笔记、会议记录、文档和知识库
- 关键词搜索和自然语言语义搜索
- 结合 BM25 全文搜索、向量语义搜索和 LLM 重排序
- 本地运行，使用 node-llama-cpp 和 GGUF 模型

**快速开始**:
```bash
# 全局安装
npm install -g @tobilu/qmd
# 或 bun
bun install -g @tobilu/qmd

# 创建集合
qmd collection add ~/notes --name notes
qmd collection add ~/Documents/meetings --name meetings
qmd collection add ~/work/docs --name docs

# 添加上下文（关键特性）
qmd context add qmd://notes "Personal notes and ideas"
qmd context add qmd://meetings "Meeting transcripts and notes"

# 生成嵌入向量
qmd embed

# 搜索
qmd search "project timeline"          # 快速关键词搜索
qmd vsearch "how to deploy"            # 语义搜索
qmd query "quarterly planning process"   # 混合 + 重排序（最佳质量）
```

**AI Agent 集成**:
```bash
# 结构化结果供 LLM 使用
qmd search "authentication" --json -n 10

# 列出所有相关文件
qmd query "error handling" --all --files --min-score 0.4

# 检索完整文档内容
qmd get "docs/api-reference.md" --full
```

**MCP 服务器**:
工具暴露:
- `query` — 带类型化子查询的搜索（lex/vec/hyde），通过 RRF + 重排序组合
- `get` — 按路径或 docid 检索文档（带模糊匹配建议）
- `multi_get` — 通过 glob 模式、逗号分隔列表或 docids 批量检索
- `status` — 索引健康和集合信息

## 运行与开发方式

**安装**:
```bash
# npm
npm install -g @tobilu/qmd

# bun (推荐)
bun install -g @tobilu/qmd

# npx (无需安装)
npx @tobilu/qmd ...
```

**快速开始**:
```bash
# 1. 创建集合
qmd collection add ~/notes --name notes

# 2. 添加上下文（关键！帮助 LLM 理解文档用途）
qmd context add qmd://notes "Personal notes and ideas"

# 3. 生成嵌入
qmd embed

# 4. 搜索
qmd search "project timeline"
qmd vsearch "how to deploy"
qmd query "quarterly planning"
```

**Claude Code 集成**:
```bash
# 通过插件市场安装
claude plugin marketplace add tobi/qmd
claude plugin install qmd@qmd

# 或手动配置 MCP
# ~/.claude/settings.json
{
  "mcpServers": {
    "qmd": {
      "command": "qmd",
      "args": ["mcp"]
    }
  }
}
```

**Claude Desktop 配置**:
```json
// ~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "qmd": {
      "command": "qmd",
      "args": ["mcp"]
    }
  }
}
```

**HTTP Transport 模式**:
```bash
# 前台运行
qmd mcp --http                    # localhost:8181

# 后台守护
qmd mcp --http --daemon
qmd mcp stop                    # 停止
```

**SDK 库使用**:
```bash
npm install @tobilu/qmd
```

```typescript
import { createStore } from '@tobilu/qmd'

const store = await createStore({
  dbPath: './index.sqlite',
  config: {
    collections: {
      docs: { path: '/path/to/docs', pattern: '**/*.md' }
    }
  }
})

const results = await store.search({ query: "authentication flow" })
console.log(results.map(r => `${r.title} (${Math.round(r.score * 100)}%)`))

await store.close()
```

## 外部接口

**CLI 命令**:
| 命令 | 功能 |
|------|------|
| `collection add` | 添加文档集合 |
| `context add` | 添加集合上下文 |
| `embed` | 生成向量嵌入 |
| `search` | 关键词搜索 |
| `vsearch` | 语义搜索 |
| `query` | 混合搜索 + 重排序 |
| `get` | 获取单个文档 |
| `multi-get` | 批量获取文档 |
| `mcp` | 启动 MCP 服务器 |

**MCP 工具**:
| 工具 | 功能 |
|------|------|
| `query` | 搜索（带 lex/vec/hyde 子查询） |
| `get` | 按路径或 docid 检索文档 |
| `multi_get` | 批量检索（glob/列表/docids） |
| `status` | 索引健康状态 |

**Agent 输出格式**:
```bash
# JSON 结构化输出
qmd search "authentication" --json -n 10

# 文件列表
qmd query "error handling" --all --files --min-score 0.4

# 完整内容
qmd get "docs/api.md" --full
```

**HTTP API** (HTTP Transport 模式):
- `POST /mcp` — MCP Streamable HTTP
- `GET /health` — 健康检查

## 数据流 / 控制流

```
用户输入: 自然语言查询
    │
    ▼
┌────────────────────────────────────────────────────────────┐
│ 1. 查询扩展 (Query Expansion)                                 │
│    - 使用 LLM 将自然语言扩展为结构化查询                      │
│    - 生成 lex (关键词) / vec (向量) / hyde (假设文档) 子查询  │
└────────────────────────────────────────────────────────────┘
    │
    ├───────────────────────────────────────────────────────┐
    ▼                                                       ▼
┌─────────────────────┐                           ┌─────────────────────┐
│ 2a. BM25 全文搜索    │                           │ 2b. 向量语义搜索     │
│                     │                           │                     │
│ - 倒排索引快速匹配   │                           │ - node-llama-cpp    │
│ - 关键词相关性评分   │                           │ - GGUF 模型本地嵌入 │
│                     │                           │ - 余弦相似度计算     │
└──────────┬──────────┘                           └──────────┬──────────┘
           │                                                  │
           └──────────────────┬───────────────────────────────┘
                              │
                              ▼
┌────────────────────────────────────────────────────────────┐
│ 3. RRF 融合 (Reciprocal Rank Fusion)                       │
│    - 合并 BM25 和向量搜索结果                              │
│    - 基于排名倒数加权融合                                  │
└────────────────────────────────────────────────────────────┘
    │
    ▼
┌────────────────────────────────────────────────────────────┐
│ 4. LLM 重排序 (Reranking)                                    │
│    - 使用本地 LLM 对候选结果重新评分                        │
│    - 考虑查询意图和文档相关性                                │
│    - 最终排序输出                                           │
└────────────────────────────────────────────────────────────┘
    │
    ▼
┌────────────────────────────────────────────────────────────┐
│ 5. 结果输出                                                 │
│    ├── 终端表格显示                                         │
│    ├── --json (结构化 JSON)                                │
│    └── --files (文件路径列表)                              │
└────────────────────────────────────────────────────────────┘
    │
    ▼
Agent / 用户获取结果
```
