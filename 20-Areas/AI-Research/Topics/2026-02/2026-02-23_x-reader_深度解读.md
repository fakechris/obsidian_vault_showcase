# x-reader - 通用内容读取器的三层架构设计

> **一句话定义**: x-reader是一个模块化通用内容读取器，采用Python CLI/库、Claude-Code Skills、MCP Server三层架构，支持从微信公众号、B站、YouTube、小红书、Telegram等7+平台自动获取和转录内容。

---

## 详细解释

### What
x-reader是一个多层可组合的内容摄取工具：
- **Layer 1 (Python CLI/Library)**: 基础内容获取，统一数据模型
- **Layer 2 (Claude-Code Skills)**: 视频/播客转录 + AI内容分析
- **Layer 3 (MCP Server)**: 将读取能力暴露为MCP工具

支持平台：微信公众号、小红书、Twitter/X、YouTube、Bilibili、Telegram、RSS、小宇宙播客、Apple Podcasts

### Why
不同内容平台有不同的反爬策略和数据格式：
- **中文生态特殊需求**: 微信公众号、小红书需要特殊处理
- **多媒体内容**: 视频需要字幕提取或Whisper转录
- **渐进式能力**: 基础用户只需CLI，高级用户可用Skills增强
- **MCP生态整合**: 通过MCP协议与Claude Desktop等工具集成

### How
**三层架构协同工作**:
```
用户发送URL
    │
    ├─ 文本内容（文章、推文、公众号）
    │   └─ Python fetcher → UnifiedContent → inbox
    │
    ├─ 视频（YouTube、Bilibili、X视频）
    │   ├─ Python fetcher → 元数据（标题、描述）
    │   └─ Video skill → 字幕或Whisper完整转录
    │
    ├─ 播客（小宇宙、Apple Podcasts）
    │   └─ Video skill → Whisper完整转录
    │
    └─ 需要分析
        └─ Analyzer skill → 结构化报告 + 行动项
```

**Fetcher回退策略**（以微信公众号为例）：
```
URL → Jina Reader尝试 → 失败则Playwright浏览器回退 → 保存session
```

## 3个重要细节

### 1. 三层架构的渐进式设计哲学
| 层级 | 功能 | 使用场景 |
|------|------|----------|
| Python CLI | URL → Markdown | 脚本化批量处理 |
| Claude Skills | 视频转录 + AI分析 | 与Claude Code对话时使用 |
| MCP Server | 工具化暴露 | Claude Desktop/other MCP客户端 |

**关键洞察**: 不同用户有不同需求，分层设计让每个人都能取所需，不强求全套安装。

### 2. 中国本土平台的特殊处理
针对微信公众号、小红书等反爬严格的平台：
- **Jina Reader优先**: 通用文本提取服务
- **Playwright回退**: 浏览器自动化模拟真实访问
- **Session持久化**: `x-reader login xhs`保存登录态，后续免登

### 3. UnifiedContent数据模型
所有平台输出统一格式：
```python
class UnifiedContent:
    title: str
    content: str
    url: str
    platform: str  # wechat, xhs, youtube, etc.
    author: str
    publish_date: datetime
    content_type: str  # text, video, podcast
    transcript: Optional[str]  # 视频/播客的转录文本
    metadata: dict  # 平台特定数据（点赞、评论数等）
```

## 2条行动建议

### 建议1: 为自己的工具设计渐进式能力层级
借鉴x-reader的三层架构：
1. **基础层**: 核心功能，最小依赖，pip即可安装
2. **增强层**: 与特定生态集成（如Claude Code Skills），可选安装
3. **协议层**: 通过开放协议（如MCP）暴露能力，让其他工具调用

### 建议2: 为中国内容平台设计专门的处理策略
针对微信公众号、小红书等平台：
- 先用轻量级服务（Jina Reader）尝试
- 失败时回退到浏览器自动化（Playwright）
- 实现登录态持久化，避免重复扫码
- 输出统一格式，降低下游处理复杂度
