# 📚 10-Knowledge: 知识层

> **"知识不是信息的堆积，而是可连接的概念网络"**

这一层是知识库的**核心知识** — 经过整理、关联、提炼的内容。

---

## 📁 目录结构

| 目录 | 用途 | 内容类型 |
|------|------|----------|
| [[Atlas/\|Atlas]] | 知识地图 | MOC、概念图谱、主题索引 |
| [[Evergreen/\|Evergreen]] | 常青笔记 | 原子化概念笔记 |
| [[Literature/\|Literature]] | 文献笔记 | 原文 + 初步标记 |
| [[Sources/\|Sources]] | 原文存储 | 原始文章归档 |

---

## 🎯 各子目录说明

### Atlas/ — 知识地图
- **MOC-Index.md**: 所有 MOC 入口
- **AI-Concepts.md**: AI 核心概念图谱
- **Agent-Patterns.md**: Agent 架构模式
- **Investment-Theses.md**: 投资主题

### Evergreen/ — 常青笔记
- 原子化概念笔记（一句话能说清一个概念）
- 高度可链接（作为知识网络的节点）
- 定期维护更新（保持"常青"）
- 命名规则: `概念名.md`

### Literature/ — 文献笔记
- 按年份/月份组织
- 原文 + 高亮 + 初步标记
- 是 Evergreen 笔记的原材料
- 格式: `YYYY/MM/DD_标题.md`

### Sources/ — 原文存储
- 从 Inbox 归档的原始文章
- 保留完整原始内容
- 用于溯源和引用

---

## 🔄 工作流

### 从 Inbox → Knowledge
1. **原始文章** → Sources/ (保留)
2. **阅读 + 标记** → Literature/ (原文 + 标记)
3. **提炼概念** → Evergreen/ (原子笔记)
4. **建立连接** → Atlas/ (知识地图)

### 从 Areas → Knowledge
- 研究过程中发现新概念 → 创建 Evergreen 笔记
- 项目经验提炼 → 创建 Atlas 模式文档

---

## 📝 笔记类型说明

| 类型 | 位置 | 特点 | 示例 |
|------|------|------|------|
| **常青笔记** | Evergreen/ | 原子化、永久价值、高度链接 | `AI-Agent.md` |
| **文献笔记** | Literature/ | 与特定来源绑定、原文保留 | `2026/03/30_khoj-ai_khoj.md` |
| **MOC** | Atlas/ | 导航、索引、全景 | `MOC-Index.md` |
